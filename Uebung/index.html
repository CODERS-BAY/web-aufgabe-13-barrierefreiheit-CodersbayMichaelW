Principles

Clarity

  • Clear Language, Flow and Hierarchy for visuelle elements.
  • It should not be possible to provoke a error.

Compactness

  • A big challenge to compine clarity and compactness with eachother.
  • Don't overflowed the user with permanent help, description, warnings and hints.

Familiarity

  • Use familiar/traditional control elements and program sequences.
  • Don't use to many new innovation.

Responsiveness

  • After a click or keystroke there should be an immediately feedback to the user.
  • Inform the user when there is longer loading times.

Constistancy

  • Buttons should have the same trigger and animation.
  • The Layout should be the same on all the pages of the Website.

Aesthetics

  • Optik is a very important aspect for an appealing Design of a Website.
  • This is the part where the user desiced if he wants to use the Website or not.
  • Small mistakes can be overseen by the user when the aesthic of the Website is appealing.

Tolerance

  • User makes mistakes and should be dealt trouble-free.
  • The Application should help/guide the user. (undo-function)
  • Security queries are not well seen by the user.

Thoughts before creating

  • It's not possible to create a 100-percent accessibility of a website.
  • Make all the Information accessible to all of your users.
  • Use established standards to meet the expectation of the users.

Who needs it?

  • Users with empty wireless mouse battery.
  • People above 50.
  • People with Tendonitis in the wrist
  • For blind people.
  • Users with a red-green weakness.
  • Users with a slow internet connection.
